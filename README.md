# 食用该仓库的一些指南
此处你看到的README文档就是我这个仓库，本项目的说明书哦！在创建仓库的适合我就选择自动创建啦，如果你由于看不懂英文，第一次创建没有点哪个按钮的话，创建好仓库再次创建新文件，命名为README.md，这样实现效果是一致的，我这里是把我这个仓库不同文件分别是什么做了一个说明，否则以后我自己看八成也不知道是什么啦

## 不同文件下👇是什么

### 前两个test1和2
1. 只是我最初建立仓库后一些尝试，
2. 先仓库内创建第一个test，看看不同按钮都是干嘛使的，由于刚开始不知道通过命名的方式是可以对文件格式进行设定的，最初只是纯文本格式的文件，
3. 最后发现与README.md不一样，了解后发现是.md是markdown文档格式，有专门的书写语法规则，于是建立了第三个文件：myself test3.md
### myself test3.md
* 是我进行markdown语法学习的实操记录，这里对于想使用Github做笔记管理，写笔记的友友要看看了，直接快速入门
* 但在实操过程中，由于我需要引用图片，大白话就是我想给我的文档加一个图片，可我没有图片文件夹📁，相对引用需要写图片位置，于是开始有了文件夹Images
### Images
1. 新建方法：与新建文件相同，只是在命名时需要加上/
2. 由于Github不允许建立空的文件夹，二次命名时我输出了.gitkeep，这是一个占位符文件
   
#### gitkeep
这个文件其实在上传图片进这个文件夹后就可以删除了，但我留存下来了，是学习记录！便于之后操作理解，我再这个文件里写了新建文件夹的方式，也是一种加深记忆的方法，食用本指南可以再次跟一遍，自己也写下来，重复记忆
